プロンプト5 — 電卓結果の表示・読み上げ問題の修正
問題: 電卓で挿入した 3,600×155\n=558,000 がメモ一覧のプレビューや読み上げ（TalkBack）でそのまま「さんぜんろっぴゃく かける ひゃくごじゅうご イコール...」のように読まれる。「計算結果: 558,000」のように変換したい。

修正方針: 挿入時にHTMLのカスタム属性（<span data-calc-expr="3600×155" data-calc-result="558000">...）で意味情報を付与し、プレビュー・読み上げ時に変換する。

Copy// 式＋結果の挿入時

richTextState.insertHtml(calcHtml)

// 結果のみ挿入時

contentPlainText の表示前に計算式パターンを変換する関数を追加:

Copyfun formatCalcForPreview(text: String): String {
    // "3,600×155\n=558,000" → "計算: 558,000" に変換
    val calcPattern = Regex("""[\d,]+[×÷+\-][\d,×÷+\-\s]*\n?=\s*([\d,]+\.?\d*)""")
    return calcPattern.replace(text) { match ->

    }
}

プロンプト7 — 蛍光ペン（ハイライト）を選択テキストに適用する
現状の問題: 蛍光ペンボタンを押すとその後に入力するテキストにハイライトが適用されるが、既に選択中のテキストには適用されない。

修正方針: compose-rich-editor の toggleSpanStyle は選択範囲がある場合はその範囲に適用し、選択範囲がない場合は以降の入力に適用する動作が期待される。SpanStyle(background = ...) を使う。

EditorScreen.kt のハイライトボタン処理:

Copy// 蛍光ペンボタンの onClick
fun onHighlightClick(color: Color) {
    val highlightStyle = SpanStyle(background = color)
    richTextState.toggleSpanStyle(highlightStyle)
}
もし現在のコードが toggleSpanStyle ではなく独自のHTML操作をしているなら、それを toggleSpanStyle(SpanStyle(background = highlightColor)) に置き換える。

確認すべきポイント:

richTextState.selection が空でない（テキストが選択されている）状態で toggleSpanStyle を呼ぶと、その選択範囲にスタイルが適用される — これが compose-rich-editor の標準動作
もし効かない場合は、ハイライトボタン押下時に選択が解除されていないか確認する。IconButton の onClick がフォーカスを奪って選択が解除される問題がある場合、ボタンに Modifier.focusProperties { canFocus = false } を付けるか、interactionSource を調整して選択を維持する
CopyIconButton(
    onClick = { onHighlightClick(selectedHighlightColor) },
    modifier = Modifier.focusProperties { canFocus = false }
) {
    Icon(Icons.Default.Highlight, "蛍光ペン")
}
