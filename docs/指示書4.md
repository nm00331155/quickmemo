# 指示: ロック画面通知のTodo専用化 + Direct Reply + 通知位置の最適化 + 初回設定ガイド

## 変更点一覧

1. 通知・ロック画面をTodo表示に特化（メモ表示は削除、「メモを編集」アクションは残す）
2. Direct Replyで「Todo追加」をロック画面から可能にする（既存の「メモ追加」は削除）
3. 通知をできるだけ上位に表示（`IMPORTANCE_DEFAULT` + 消音）
4. 折りたたみ状態でも3行のTodo内容を表示
5. 初回起動時にロック画面通知の設定ガイドダイアログを表示
6. Todoのドラッグ並べ替えが動作しない問題の修正
7. Todo日付アイコンの変更

### B. buildNotification() をTodo専用に書き換え

`buildNotification()` メソッド全体を以下で置換:

```kotlin
private suspend fun buildNotification(): Notification {
    val db = QuickMemoDatabase.getInstance(applicationContext)
    val todoDao = db.todoDao()

    // Todoデータ取得
    val uncheckedItems = try {
        todoDao.getUncheckedItemsSync()
    } catch (e: Exception) {
        Log.e(TAG, "Failed to load unchecked todos", e)
        emptyList()
    }

    val checkedCount = try {
        todoDao.getCheckedCountSync()
    } catch (_: Exception) {
        0
    }

    val totalCount = uncheckedItems.size + checkedCount

    Log.d(TAG, "Building notification: unchecked=${uncheckedItems.size}, checked=$checkedCount, total=$totalCount")

    // --- 折りたたみ表示 (3行でTodo内容を表示) ---
    val collapsedView = RemoteViews(packageName, R.layout.notification_collapsed)

    val summaryText = if (totalCount > 0) {
        "☑ Todo $checkedCount/$totalCount 完了"
    } else {
        "☑ Todoなし"
    }
    collapsedView.setTextViewText(R.id.notification_summary, summaryText)

    val lineIds = listOf(
        R.id.notification_line1,
        R.id.notification_line2,
        R.id.notification_line3,
    )
    val displayItems = uncheckedItems.take(3)
    for (i in lineIds.indices) {
        if (i < displayItems.size) {
            collapsedView.setTextViewText(lineIds[i], "☐ ${displayItems[i].text}")
            collapsedView.setViewVisibility(lineIds[i], android.view.View.VISIBLE)
        } else {
            collapsedView.setViewVisibility(lineIds[i], android.view.View.GONE)
        }
    }

    // --- 展開表示 (5件表示) ---
    val expandedView = RemoteViews(packageName, R.layout.notification_expanded)
    expandedView.setTextViewText(R.id.notification_header, "☑ Todo ($checkedCount/$totalCount 完了)")

    val todoTextIds = listOf(
        R.id.notification_todo_1,
        R.id.notification_todo_2,
        R.id.notification_todo_3,
        R.id.notification_todo_4,
        R.id.notification_todo_5,
    )
    val expandedItems = uncheckedItems.take(5)
    for (i in todoTextIds.indices) {
        if (i < expandedItems.size) {
            expandedView.setTextViewText(todoTextIds[i], "☐ ${expandedItems[i].text}")
            expandedView.setViewVisibility(todoTextIds[i], android.view.View.VISIBLE)
        } else {
            expandedView.setViewVisibility(todoTextIds[i], android.view.View.GONE)
        }
    }

    // ピン留めメモの行は非表示にする
    expandedView.setViewVisibility(R.id.notification_pinned_memo, android.view.View.GONE)

    val remainingCount = uncheckedItems.size - 5
    if (remainingCount > 0) {
        expandedView.setTextViewText(R.id.notification_footer, "他 ${remainingCount}件 → タップして全件表示")
    } else if (uncheckedItems.isEmpty()) {
        expandedView.setTextViewText(R.id.notification_footer, "すべて完了！ →")
    } else {
        expandedView.setTextViewText(R.id.notification_footer, "タップして開く →")
    }
    expandedView.setViewVisibility(R.id.notification_footer, android.view.View.VISIBLE)

    // --- PendingIntent ---
    // メインタップ → Todo画面を開く
    val openTodoIntent = Intent(this, MainActivity::class.java).apply {
        action = QuickMemoIntents.ACTION_OPEN_TODO
        flags = Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_CLEAR_TOP
    }
    val openTodoPending = PendingIntent.getActivity(
        this, 100, openTodoIntent,
        PendingIntent.FLAG_IMMUTABLE or PendingIntent.FLAG_UPDATE_CURRENT,
    )

    // --- Direct Reply: ロック画面からTodo追加 ---
    val addTodoReplyKey = "key_add_todo_text"
    val remoteInputTodo = androidx.core.app.RemoteInput.Builder(addTodoReplyKey)
        .setLabel("Todoを入力...")
        .build()

    val addTodoBroadcastIntent = Intent(this, NotificationActionReceiver::class.java).apply {
        action = NotificationActionReceiver.ACTION_ADD_TODO
    }
    val addTodoPending = PendingIntent.getBroadcast(
        this,
        200,
        addTodoBroadcastIntent,
        PendingIntent.FLAG_MUTABLE or PendingIntent.FLAG_UPDATE_CURRENT,
    )
    val addTodoAction = NotificationCompat.Action.Builder(
        0,
        "Todo追加",
        addTodoPending,
    ).addRemoteInput(remoteInputTodo)
        .setAllowGeneratedReplies(false)
        .build()

    // メモ編集ボタン（Direct Replyではなく通常のアクション）
    val openEditorIntent = Intent(this, MainActivity::class.java).apply {
        action = QuickMemoIntents.ACTION_OPEN_EDITOR
        flags = Intent.FLAG_ACTIVITY_NEW_TASK
    }
    val openEditorPending = PendingIntent.getActivity(
        this, 101, openEditorIntent,
        PendingIntent.FLAG_IMMUTABLE or PendingIntent.FLAG_UPDATE_CURRENT,
    )

    // --- 通知構築 ---
    return NotificationCompat.Builder(this, CHANNEL_ID)
        .setSmallIcon(R.drawable.ic_notification_memo)
        .setOngoing(true)
        .setPriority(NotificationCompat.PRIORITY_DEFAULT)  // ← LOW→DEFAULT
        .setCategory(NotificationCompat.CATEGORY_REMINDER)
        .setVisibility(NotificationCompat.VISIBILITY_PUBLIC)
        .setCustomContentView(collapsedView)
        .setCustomBigContentView(expandedView)
        .setStyle(NotificationCompat.DecoratedCustomViewStyle())
        .setContentIntent(openTodoPending)
        .addAction(addTodoAction)                                  // Todo追加（Direct Reply）
        .addAction(0, "メモを編集", openEditorPending)              // メモ編集（通常アクション）
        .build()
}
```

## 2. `NotificationActionReceiver.kt` の修正（Todo専用に簡略化）

メモ追加の処理を削除し、Todo追加のみに絞る:

```kotlin
package com.quickmemo.app.receiver

import android.content.BroadcastReceiver
import android.content.Context
import android.content.Intent
import android.util.Log
import androidx.core.app.RemoteInput
import com.quickmemo.app.data.local.database.QuickMemoDatabase
import com.quickmemo.app.data.local.entity.TodoItemEntity
import com.quickmemo.app.service.QuickMemoForegroundService
import com.quickmemo.app.widget.WidgetUpdateDispatcher
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.launch
import java.util.UUID

class NotificationActionReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == ACTION_ADD_TODO) {
            handleAddTodo(context, intent)
        }
    }

    private fun handleAddTodo(context: Context, intent: Intent) {
        val remoteInput = RemoteInput.getResultsFromIntent(intent) ?: return
        val inputText = remoteInput.getCharSequence("key_add_todo_text")?.toString()?.trim()
        if (inputText.isNullOrBlank()) return

        Log.d(TAG, "Adding Todo from lock screen: $inputText")

        val pendingResult = goAsync()
        CoroutineScope(Dispatchers.IO).launch {
            try {
                val db = QuickMemoDatabase.getInstance(context)
                val todoDao = db.todoDao()

                val maxOrder = todoDao.getMaxSortOrder(0) ?: -1
                val newItem = TodoItemEntity(
                    id = UUID.randomUUID().toString(),
                    tabId = 0,
                    text = inputText,
                    checked = 0,
                    dueDate = null,
                    sortOrder = maxOrder + 1,
                    createdAt = System.currentTimeMillis(),
                    checkedAt = null,
                )
                todoDao.insert(newItem)
                Log.d(TAG, "Todo added successfully: $inputText")

                // 通知 + ウィジェットを更新
                QuickMemoForegroundService.refreshFromOutside(context)
                WidgetUpdateDispatcher.updateTodoWidgets(context)
            } catch (e: Exception) {
                Log.e(TAG, "Failed to add todo", e)
            } finally {
                pendingResult.finish()
            }
        }
    }

    companion object {
        const val ACTION_ADD_TODO = "com.quickmemo.app.ACTION_ADD_TODO_FROM_NOTIFICATION"
        private const val TAG = "QM_NOTIF_ACTION"
    }
}
```

## 5. Todoのドラッグ並べ替えが動作しない問題の修正

### 原因の調査ポイント

`sh.calvin.reorderable` ライブラリでドラッグ並べ替えが動作しない典型的な原因:

#### A. `LazyColumn` の `key` 設定漏れ

`LazyColumn` の `items` に `key` が設定されていないと reorderable が動作しない。

```kotlin
items(
    items = todoItems,
    key = { it.id },  // ← これが必須
) { item ->
```

#### B. `ReorderableItem` のラップ漏れ

各アイテムを `ReorderableItem` でラップしていない可能性:

```kotlin
ReorderableItem(
    reorderableState = reorderableState,
    key = item.id,
) { isDragging ->
    // アイテムのUI
    Row(...) {
        // ... チェックボックス、テキスト、日付、ドラッグハンドル
        IconButton(
            modifier = Modifier.draggableHandle(),  // ← これが必須
            onClick = {},
        ) {
            Icon(Icons.Default.DragHandle, contentDescription = "並べ替え")
        }
    }
}
```

#### C. `rememberReorderableLazyListState` の設定

```kotlin
val reorderableState = rememberReorderableLazyListState(lazyListState) { from, to ->
    // items リストの並べ替え
    todoItems = todoItems.toMutableList().apply {
        add(to.index, removeAt(from.index))
    }
}
```

#### D. ドラッグ完了後のDB保存

ドラッグ終了時（`onDragEnd` 等）に `sortOrder` を更新してDBに保存する必要がある:

```kotlin
// reorderableState のコールバック内、または LaunchedEffect で
// ドラッグ完了後に新しい順序をDBに保存
scope.launch {
    todoItems.forEachIndexed { index, item ->
        todoRepository.updateSortOrder(item.id, index)
    }
}
```

### 具体的な修正

Todo画面のソースファイル（`TodoScreen.kt` 等）を確認し、上記A〜Dのいずれかが欠けている箇所を修正すること。特にドラッグハンドル（≡アイコン）に `.draggableHandle()` modifier が付いていることを確認。

**ドラッグハンドルのアイコン**: 現在 `=` テキストまたは `DragHandle` アイコンが使われているはず。`Icons.Default.DragHandle` (`≡`) が Material Icons にない場合は `Icons.Default.Menu` で代用するか、カスタムアイコンを使用。

## 6. Todo日付アイコンの変更

現在の日付設定アイコン（カレンダーアイコン）を変更する。

### 方法A: Material Icons の別シンボルに差し替え

Todo画面のアイテム行にある日付アイコンを探し、以下のいずれかに変更:

```kotlin
// 現在（微妙と感じているアイコン）
Icon(Icons.Default.CalendarToday, ...)

// 候補1: カレンダーに時計が付いたもの
Icon(Icons.Default.EditCalendar, contentDescription = "期日設定")

// 候補2: イベント系
Icon(Icons.Default.Event, contentDescription = "期日設定")

// 候補3: スケジュール系
Icon(Icons.Outlined.EventNote, contentDescription = "期日設定")

// 候補4: アラーム付き
Icon(Icons.Outlined.Alarm, contentDescription = "期日設定")
```

### 方法B: カスタムSVG/PNG を使用する場合

1. SVGまたはPNGファイルを用意
2. Android Studio で `res/drawable/` → 右クリック → New → Vector Asset → Local file
3. 変換された `ic_custom_calendar.xml` を配置
4. コードで使用:

```kotlin
Icon(
    painter = painterResource(R.drawable.ic_custom_calendar),
    contentDescription = "期日設定",
)
```

ユーザーがカスタムアイコンを提供する場合はこの方法で差し替え可能。提供がなければ `Icons.Outlined.Event` に変更すること。
