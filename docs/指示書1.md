## 第2章 ── 合意済み変更点（この章が作業指示）

### 2.1 【削除】AI機能の全面撤去

v1.0ではAI機能を**搭載しない**。以下を全て削除すること:

- `ai/` パッケージ配下の全ファイル（AiEngine系, AiService, AiUsage等）
- ModelManager, ModelDownloadWorker
- AI BottomSheet UI（AiBottomSheet コンポーネント）
- Qwen2.5/Qwen3 関連コード、`de.kherud:llama` 依存
- Gemini Nano 関連コード、`com.google.mlkit.genai` 依存
- `com.google.android.apps.aicore.service.BIND_SERVICE` パーミッション
- DataStore キー: `ai_enabled`, `selected_ai_engine`
- ローカルモデルファイル: `ai_models/qwen2.5-1.5b-instruct-q4_k_m.gguf` (1.04 GB)
- エディタのツールバー・メニューからAI関連ボタンを除去
- 設定画面からAIエンジン選択セクションを除去

**将来対応**: Gemini Nano が日本語を十分にサポートした時点、またはユーザー基盤が十分になりクラウドAPIのサーバー代理を運用できる時点で再検討。ツールバーのAIボタン位置には「今後のアップデートで対応予定」のToastを出してもよい（任意）。

### 2.2 【削除】チェックリスト機能の撤去

メモエディタ内のチェックリスト機能を削除:

- メインバーの「☑チェック」ボタンを除去
- `MemoEntity.isChecklist` フラグは DB カラムとして残すが（マイグレーション回避）、UIでは使用しない
- チェックリストモードでの新規メモ作成（ウィジェット・通知バー）を除去
- ホーム画面のチェックリスト表示プレビューを除去

**理由**: チェックリストの役割は独立した Todo 機能に移行済み。

### 2.3 【削除】範囲選択モード (selection_mode) の撤去

- `memo_toolbar_selection_mode` は**無効化・削除**する
- DataStore からキーを削除、設定UIからも除去

**理由**: compose-rich-editor (rc13) のテキスト選択は、日本語テキストで範囲選択すると文全体が広く選択されてしまい、微調整が非常に困難。これはライブラリ側の既知の制約であり、カスタム選択モードで対処するのは現実的でない。通常のOS標準テキスト選択（ロングプレス→ハンドルドラッグ）に委ねる。

### 2.4 【削除】画像添付機能の撤去

- メインバーの「📷画像」ボタンを除去
- 画像ピッカー / カメラ撮影による画像挿入機能を除去
- `android.permission.CAMERA`, `android.permission.READ_MEDIA_IMAGES` はOCR用途で維持

**理由**: リッチテキストエディタ内の画像管理は複雑さが大きく、v1.0のスコープ外。

### 2.5 【変更】ナビゲーション構造

BottomNavigation または TopBar にメモ/Todoの切替を配置:

- DataStore キー `todo_placement` で `TOP_BAR` / `BOTTOM_NAV` を切替可能（既に実装済み）
- デフォルト: `TOP_BAR`
- 設定画面（一般タブ）でユーザーが変更可能
- NavHost ルート: `memo_list`, `todo`
- 画面遷移アニメーション: 200ms cross-fade

### 2.6 【変更】メモ編集ツールバー（メインバー）

旧仕様を破棄し、以下の構成に変更:

[文字サイズ] [太字] [斜体] [蛍光ペン] [文字色] [日付挿入] [電卓] [OCR] [全文コピー] [共有] [⋯その他]


| ボタン | 機能 | 備考 |
|---|---|---|
| 文字サイズ | 3段階 (小14sp/中18sp/大24sp) | SegmentedButton風 |
| 太字 (B) | `toggleSpanStyle(fontWeight=Bold)` | トグル |
| 斜体 (I) | `toggleSpanStyle(fontStyle=Italic)` | トグル |
| 蛍光ペン | テキスト背景色を複数色から選択 | カラーパレットをポップアップ表示 |
| 文字色 | テキスト前景色を7色から選択 | 現行のサブバーから移動 |
| 日付挿入 | タップ: 現在日付即挿入 / 長押し: DatePicker | 既存維持 |
| 電卓 | 簡易電卓ダイアログ（四則演算 + 括弧、結果を本文に挿入可能） | 動作不良を修正対象 |
| OCR | カメラ→ML Kit 日本語テキスト認識→本文に挿入 | 無料・オンデバイス |
| 全文コピー | `contentPlainText` をクリップボードにコピー | 新規追加 |
| 共有 | ACTION_SEND text/plain | 既存維持 |
| ⋯その他 | ピン留め、ロック、メモの色、ゴミ箱、メモ情報 | AI項目を除去 |

補足:
- 番号付きリスト機能と表機能は v1.0 スコープ外として削除する
- 旧バージョンで保存された表ブロックは、読み込み時にテキストへ変換して内容を保持する

**各ツールバーボタンは設定画面（メモタブ）で個別にON/OFF可能。** DataStore キー `memo_toolbar_*` で管理。

**旧仕様のサブバー（Aaトグル）は廃止。** 全ての書式ボタンをメインバーに統合し、横スクロールで対応。

### 2.7 【変更】Undo / Redo

- メインバーまたはトップバーに Undo (↶) / Redo (↷) ボタンを配置
- `RichTextState` の undo/redo API を使用
- compose-rich-editor のバージョンが undo 未対応の場合は、`contentHtml` の履歴スタック（最大20件）で自前実装

### 2.8 【変更】設定画面 ── 3タブ構成

単一ページから **3タブ構成** に変更:

#### タブ1: 一般
- テーマ（ライト / ダーク / システム）
- クイック入力通知（ON/OFF）
- 生体認証（アプリ起動時認証 ON/OFF）
- バックアップ / 復元（JSON形式、全メモ・Todo・設定を含む）
- ゴミ箱（→ゴミ箱画面へ遷移）
- プレミアム / 課金管理
- ナビゲーション配置（TOP_BAR / BOTTOM_NAV）
- バージョン / ライセンス / お問い合わせ

#### タブ2: メモ
- 新規メモのデフォルト色（7色）
- 文字数カウント表示（ON/OFF）
- 日付挿入フォーマット選択
- ツールバー項目の個別 ON/OFF

#### タブ3: Todo
- リマインダー通知（ON/OFF）
- リマインダータイミング（○時間前、○分前、カスタム）
- タブ名編集（3タブ、各10文字以内）

### 2.9 【変更】課金モデルの拡張

旧仕様（広告除去 ¥300 の買い切り1種のみ）から以下に拡張:

### 2.10 【変更】翻訳機能（Pro / 翻訳パック購入者のみ）

- ML Kit Translation (オンデバイス)
- `en_ja` モデル（約30 MB、初回ダウンロード）
- エディタのメニューまたはツールバーから呼び出し
- Pro未購入の場合は「プレミアム機能です」と購入誘導

### 2.11 【変更】TTS 読み上げ機能

- Android TTS API (`android.speech.tts.TextToSpeech`) を使用
- `contentPlainText` を読み上げ
- 速度: 0.5x ～ 2.0x（設定で調整）
- ピッチ: 調整可能
- エディタのメニューから呼び出し

### 2.12 【維持】Todo 機能

現行実装を維持。主要仕様:

- 3つの固定タブ（タブ名はユーザーが設定で変更可能、10文字以内）
- アイテム構成: チェックボックス → テキスト → カレンダー → ドラッグハンドル
- 完了アイテムは「完了済み (n)」折りたたみセクションに自動移動
- ドラッグ並べ替え: `sh.calvin.reorderable` ライブラリ
- リマインダー: WorkManager (`TodoReminderWorker`)、設定可能なリードタイム

### 2.13 【★目玉機能】ロック画面への Todo / メモ表示

**3つのアプローチを併用して最大限のカバレッジを実現:**

#### アプローチA: ロック画面ウィジェット（Android 16 QPR2+ 対応端末）

Android 16 QPR2（2025年12月リリース済み）でPixel端末にロック画面ウィジェットが復活。既存の4つの Glance ウィジェット（特に TodoWidgetReceiver と MemoViewWidgetReceiver）はそのまま ロック画面で利用可能。

**対応すべきこと:**
- ウィジェットが `WIDGET_CATEGORY_NOT_KEYGUARD` でオプトアウトしていないことを確認（デフォルトで許可されるので、明示的にオプトアウトしなければOK）
- ロック画面での表示に最適化: 4×3 セルサイズ対応、ダイナミックカラー対応、ダイナミックリサイズ対応
- ロック画面からActivityを起動する場合は `android:showWhenLocked="true"` を設定するか、認証を要求
- `appwidget-provider` XML に `widgetCategory="home_screen|keyguard"` を明示的に追加

#### アプローチB: カスタム常駐通知（全Android 8.0+ 対応）

既存の `QuickMemoForegroundService` を拡張して、ロック画面にTodo/メモを表示:

- `Notification.VISIBILITY_PUBLIC` でロック画面に常時表示
- `RemoteViews` を使ったカスタムレイアウト:
  - **Todo モード**: 未完了Todoアイテムを最大5件表示（チェックボックス付き、タップで完了切替）
  - **メモモード**: 最新のピン留めメモのタイトル＋プレビュー表示
  - **切替**: DataStore キー `notification_quick_input_content_mode` で `TODO_ONLY` / `MEMO_ONLY` / `BOTH` を選択
- Expanded RemoteViews（展開時）で詳細表示
- アクションボタン: 「新規メモ」「新規Todo」

#### アプローチC: ストア訴求 & ユーザー案内

- 設定画面に「ロック画面に表示する方法」のガイドセクションを追加
- Android 16 QPR2+ 端末: 「設定 > ディスプレイ > ロック画面 > ウィジェット」への誘導
- それ以外: 「通知をロック画面に表示する設定」への誘導
- Google Play ストアの説明文で「ロック画面でTodoが確認できる」を目玉としてアピール

### 2.14 【維持】その他の現行機能

以下は変更なし:

### 3.3 画面構成と遷移

アプリ起動 → ① ホーム（メモ一覧） ├─ [BottomNav or TopBar] → ⑦ Todo画面 ├→ ② メモ編集（FAB or カードタップ） ├→ ⑤ 検索（FTS4） └→ ⑥ 設定（3タブ） └→ ゴミ箱
