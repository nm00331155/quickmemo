プロンプト1 — 文字数カウント・蛍光ペンがリアルタイム反映されない（根本原因共通）
原因分析: compose-rich-editor の RichTextState は内部で annotatedString を Compose State として保持している。ツールバーボタンで toggleSpanStyle を呼んだ際やテキスト変更時に、文字数カウント用の変数が再計算されていない。おそらく現在のコードは richTextState.toHtml() や richTextState.toText() を「保存時」にしか呼んでおらず、エディタ内のリアルタイム変更を監視していない。

修正方針: snapshotFlow で RichTextState の annotatedString を監視し、変更があるたびに文字数と蛍光ペン状態を即座に反映する。

EditorScreen.kt の修正:

Copy// ① richTextState の annotatedString を snapshotFlow で監視
val annotatedString = richTextState.annotatedString

// ② 文字数をリアルタイムで derivedStateOf で算出（snapshotFlow不要、直接参照で再Compose）
val charCount by remember {
    derivedStateOf {
        richTextState.annotatedString.text.length
    }
}

// ③ 文字数表示部分に charCount を使う
if (showCharCount) {
    Text(
        text = "${charCount}文字",
        style = MaterialTheme.typography.bodySmall,
        color = MaterialTheme.colorScheme.onSurfaceVariant,
    )
}
もし現在のコードが var charCount by remember { mutableStateOf(0) } のように独立した変数に保持し、onValueChange 的なコールバック内でのみ更新しているなら、それが原因。compose-rich-editor は通常の TextField の onValueChange を直接提供しないため、richTextState.annotatedString.text.length を直接UI内で参照することでComposeの再構成が自動的にトリガーされる。

蛍光ペンの修正:

Copy// ツールバーの蛍光ペンボタン
IconButton(
    onClick = {
        richTextState.toggleSpanStyle(
            SpanStyle(background = selectedHighlightColor)
        )
    },
    // ★ 重要: ボタンクリックでエディタの選択範囲が解除されないようにする
    modifier = Modifier.focusProperties { canFocus = false },
    interactionSource = remember { MutableInteractionSource() },
) {
    Icon(Icons.Default.Highlight, "蛍光ペン")
}
ポイント: Modifier.focusProperties { canFocus = false } を全てのツールバーボタン（太字、イタリック、蛍光ペン等）に付けること。これにより、ボタンタップ時にRichTextEditorからフォーカスが奪われず、選択範囲が維持される。toggleSpanStyle は選択範囲があればその範囲に適用、なければ以降の入力に適用する仕様。

同じ原因のもう一つの確認: もし richTextState の変更を LaunchedEffect で監視してHTML保存している場合、その key が richTextState.annotatedString を含んでいるか確認。含んでいなければスタイル変更がトリガーされない。

Copy// 自動保存の監視も annotatedString を参照
LaunchedEffect(Unit) {
    snapshotFlow { richTextState.annotatedString }
        .debounce(500)
        .collect {
            // 自動保存 + 文字数更新
            viewModel.saveContent(richTextState.toHtml())
        }
}

バックアップマネージャー — MemoBackupManager.kt:

Copyclass MemoBackupManager @Inject constructor(
    private val backupDao: MemoBackupDao,
) {
    suspend fun saveRealtimeBackup(memoId: Long, title: String, contentHtml: String) {
        backupDao.insert(MemoBackupEntity(memoId = memoId, title = title, contentHtml = contentHtml, backupType = "REALTIME"))
        backupDao.pruneRealtimeBackups(memoId)
    }

    suspend fun savePeriodicBackup(memoId: Long, title: String, contentHtml: String, type: String) {
        val maxGenerations = when (type) {
            "1MIN" -> 4
            "5MIN" -> 5
            else -> return
        }
        backupDao.insert(MemoBackupEntity(memoId = memoId, title = title, contentHtml = contentHtml, backupType = type))
        pruneOldBackups(memoId, type, maxGenerations)
    }

    private suspend fun pruneOldBackups(memoId: Long, type: String, maxGenerations: Int) {
        val backups = backupDao.getBackupsByType(memoId, type)
        if (backups.size > maxGenerations) {
            val toDelete = backups.drop(maxGenerations).map { it.id }
            backupDao.deleteByIds(toDelete)
        }
    }
}

Worker — AppBackupWorker.kt:

Copyclass AppBackupWorker(
    context: Context,
    params: WorkerParameters,
) : CoroutineWorker(context, params) {

    override suspend fun doWork(): Result {
        return try {
            val dbFile = applicationContext.getDatabasePath("quickmemo.db")
            val backupDir = File(applicationContext.filesDir, "app_backups")
            backupDir.mkdirs()

            // タイムスタンプ付きバックアップファイル名
            val timestamp = SimpleDateFormat("yyyyMMdd_HHmmss", Locale.US).format(Date())
            val backupFile = File(backupDir, "quickmemo_backup_$timestamp.db")

            // Room checkpoint (WALモードの場合)
            val db = QuickMemoDatabase.getInstance(applicationContext)
            db.openHelper.writableDatabase.execSQL("PRAGMA wal_checkpoint(FULL)")

            // ファイルコピー
            dbFile.copyTo(backupFile, overwrite = true)
            // WALファイルもバックアップ（存在する場合）
            val walFile = File(dbFile.path + "-wal")
            if (walFile.exists()) {
                walFile.copyTo(File(backupFile.path + "-wal"), overwrite = true)
            }

            // 世代管理: 古いバックアップを削除
            pruneOldBackups(backupDir)

            Log.d("QM_BACKUP", "App backup created: ${backupFile.name}")
            Result.success()
        } catch (e: Exception) {
            Log.e("QM_BACKUP", "Backup failed", e)
            Result.retry()
        }
    }

    private fun pruneOldBackups(backupDir: File) {
        val maxGen = 10 // DataStoreから取得してもよいが、Worker内では簡略化
        val backups = backupDir.listFiles { f -> f.name.startsWith("quickmemo_backup_") && f.name.endsWith(".db") }
            ?.sortedByDescending { it.name }
            ?: return

        if (backups.size > maxGen) {
            backups.drop(maxGen).forEach { file ->
                file.delete()
                File(file.path + "-wal").delete()
                File(file.path + "-shm").delete()
            }
        }
    }
}

Copyfun scheduleAppBackup(context: Context, hour: Int = 0, minute: Int = 0) {
    val now = Calendar.getInstance()
    val target = Calendar.getInstance().apply {
        set(Calendar.HOUR_OF_DAY, hour)
        set(Calendar.MINUTE, minute)
        set(Calendar.SECOND, 0)
        if (before(now)) add(Calendar.DAY_OF_MONTH, 1)

    val initialDelay = target.timeInMillis - now.timeInMillis

    val workRequest = PeriodicWorkRequestBuilder<AppBackupWorker>(
        1, TimeUnit.DAYS
    )
        .setInitialDelay(initialDelay, TimeUnit.MILLISECONDS)
        .addTag("app_backup")
        .build()

    WorkManager.getInstance(context).enqueueUniquePeriodicWork(
        "app_backup_daily",
        ExistingPeriodicWorkPolicy.UPDATE,
        workRequest,
    )
}
